# sales_ai 대시보드 정책서

## 1. 개요

- **목적**: 물류 영업 담당자가 타겟 브랜드를 한눈에 파악
- **사용자**: 영업팀
- **핵심 지표**: Pain Score (네이버 쇼핑 + DataLab 기반)

---

## 2. 화면 구성

### 2.1 상단 요약 카드

| 카드 | 설명 |
|---|---|
| 전체 분석 브랜드 수 | 현재 분석된 브랜드 총 개수 |
| 평균 Pain Score | 전체 브랜드 Pain Score 평균 |
| Star 등급 브랜드 수 | Pain Score 80 이상 브랜드 수 |
| 신규 발굴 브랜드 수 | 이전 분석 대비 새로 진입한 브랜드 수 |

### 2.2 BCG 매트릭스 차트

- **X축**: 검색 트렌드 (DataLab 기반 성장률)
- **Y축**: Pain Score
- **4분면 구분**

| 분면 | 조건 |
|---|---|
| Star | X 높음 / Y 높음 |
| Cash Cow | X 낮음 / Y 높음 |
| Question Mark | X 높음 / Y 낮음 |
| Dog | X 낮음 / Y 낮음 |

- 브랜드별 점(dot)으로 표시
- 점 클릭 시 브랜드 상세 정보 표시

### 2.3 브랜드 목록 테이블

**컬럼**

| 컬럼명 | 설명 |
|---|---|
| 브랜드명 | 브랜드 이름 |
| 카테고리 | 상품 카테고리 |
| Pain Score | 최종 산출 점수 (0~100) |
| 택배사 | 주 사용 택배사 |
| 출고지 | 상품 출고 지역 |
| 등급 | Star / Cash Cow / Question Mark / Dog |
| 최종분석일 | 데이터 수집 기준일 |

**필터**
- 등급별 (Star / Cash Cow / Question Mark / Dog)
- 카테고리별
- Pain Score 범위 (슬라이더)

**정렬**
- 기본: Pain Score 내림차순

**페이지네이션**
- 20건 / 페이지

### 2.4 비즈니스 규칙 안내 패널

대시보드 하단에 Pain Score 산출 기준을 언제든 확인할 수 있는 상시 안내 패널을 제공합니다.

**구성**

| 구성 요소 | 설명 |
|---|---|
| Pain Score 계산 공식 | `성장속도 × 0.4 + CS문제 × 0.4 + SKU복잡도 × 0.2` 공식 표시 |
| 등급 기준 카드 4개 | Star / Cash Cow / Question Mark / Dog 각 등급의 점수 범위 및 영업 의미 |

**위치**: 브랜드 목록 테이블 하단 (항상 노출)

**UX 의도**: 영업 담당자가 Pain Score나 등급 의미가 궁금할 때 별도 페이지 이동 없이 즉시 확인 가능

---

## 3. 비즈니스 규칙

### Pain Score 등급 기준

| 등급 | Pain Score 범위 | 의미 |
|---|---|---|
| ⭐ Star | 80 이상 | 최우선 영업 타겟 |
| 🐄 Cash Cow | 60 ~ 79 | 안정적 영업 대상 |
| ❓ Question Mark | 40 ~ 59 | 모니터링 필요 |
| 🐕 Dog | 40 미만 | 영업 우선순위 낮음 |

> Pain Score = 성장속도 × 0.4 + CS문제 × 0.4 + SKU복잡도 × 0.2
